Loaders

All loaders implement the ILoader interface. Loaders are self configuring. If a required property is not defined, and exception is thrown.

public interface ILoader {

    public void configure(Map map) throws XGenException;

    public Object loadModel() throws XGenException;

}

Configuration

Loaders uses properties defined by the Ant task to configure itself.

loader="workzen.xgen.loader.JavaModelLoader"
xmlFilePath="${input.file}"

The loader classname must be defined in order for XGen to instantiate the class. All properties defined in the task are passed to the loader as a Map. The JavaModelLoader uses an xml file for input.
