Processors

All processors implement the IProcessor interface. Processors are self configuring. If a required property is not defined, and exception is thrown.

public interface IProcessor {

    public void configure(Map map) throws XGenException;

    public void process(Object model) throws XGenException;

}

VelocityProcessor

The VelocityProcessor uses properties defined by the Ant task to configure itself.

processor="workzen.xgen.engine.VelocityProcessor"
 templateBasePath="${basedir}/src/workzen/xgen"
 templatePath="templates/persistence/manager"
 template="control_table.vm"
 outputPath="${output.dir}"
 outputFile="table.out"

The processor classname must be defined in order for XGen to instantiate the processor. All properties defined in the task are passed to the processor and will be added to the VelocityContext. In addition to configuring the engine, properties can be extracted from anywhere within a template.

Two path properties are defined in the task.

  • templateBasePath is used by Velocity to search the filesystem for the template
  • templatePath is used by Velocity to search the classPath and jar files for the template.

The order of the search is "file,classpath". This allows the developer to customize templates as needed from the filesystem.
