Quick Start

  1. Install the supporting libraries:
  • Velocity
  • Ant
  • dom4j
  • JUnit
  • JDBC driver
  1. Unzip Xgen onto your system.
  2. Create an xml map file and place it into the xml directory: .../xgen/project/xgen/xml
  3. Create (or copy from .../xgen/project/gen/conf ) an ant property file and place it in: .../xgen/ant
  4. Specify the property file to run in .../xgen/ant/build.properties
  5. Run "ant make-source" from .../xgen/ant

Overview

Xgen is a Velocity Texen Task. It reads a xml file into a data structure which gets merged with a Velocity template, producing source code. The map file  defines the elements of your classes: the package, beans (classes) within the package, and fields (attributes) within the bean.

Ant has four tasks which control the project:

ant make-project

This ant task builds the Xgen source code, generates the javadocs, jars the class files, and copies the jar into the lib directory. The lib directory is in the classpath and will be accessed when making the source.

ant make-source

Ant make-source runs the Velocity texen task. Texen reads a template file (control.vm and template.vm),  and merges it with the "context". Inside the context are some simple java datastructures that contain the elements "package, beans and fields" specified in xml map file.

ant make-test

This task builds the test program, generates the javadocs, jars the class files, and copies the jar into the lib directory. The lib directory is in the classpath and will be accessed when running the test program.

ant run-program

This task runs the test program. The test program uses the TypeBean.class to run CRUD operations on the database. The database parameters must be set up in the .../xgen/ant/db.properties file.

About

Xgen was designed on a Windows workstation. There will be CR/LFs when transferred to a unix system. All binaries can be rebuilt with the ant project, allowing an ftp in ascii mode to clean up the extra CR.
