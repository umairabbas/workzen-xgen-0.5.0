JDBC Manager Class

The generated JDBC classes can persist any Java class with standard accessors, and "id" fields. It can also persist contained classes and references. See the examples.

Goal

The goal of this persistence framework is to produce human readable JDBC code that can be managed at a developer level.  Many frameworks exist to persist Java classes via xml schema definitions. For example:

  • Apache Torque
  • Apache OJB

Because persistence behavior is complex, I have found it easier to control the behavior of contained relationships via code rather than complex xml schema manipulation. This project can generate 95% of your JDBC code.

Table Level Persistence

All persistable classes have a corresponding SQL table. Table columns relate directly to Java class "fields".

Manager Classes

Each persistable class also has a "manager" class that manages the contained relationships. You can control, through code comments, which contained relationships get persisted, or deleted.

ID Fields

Each class needing persistence must include a numeric "id" field. This id field corresponds to the primary key field of the table row. In addition, each reference needing persistence, must also have an "id" field. This will be populated by the foreign-key. This is similar to how OJB and Torque persist their classes.
